# gesture-control
# 🖐️ Управление мышью жестами руки через камеру

Проект позволяет управлять курсором и кликать левой кнопкой мыши с помощью жестов руки, отслеживаемых через веб-камеру. Приложение создаёт **прозрачное окно без рамок**, которое всегда поверх всех окон, и отображает скелет руки в реальном времени.

---

## ✨ Возможности

- **Движение мыши** – удерживайте **указательный + большой пальцы** вместе, и курсор будет плавно следовать за движением пальца.
- **Левый клик** – быстрое касание (тап) теми же пальцами (менее 0.3 секунды) имитирует нажатие ЛКМ.
- **Отслеживание первой руки** – программа запоминает первую появившуюся руку (правую или левую) и управляет только ей, игнорируя другие руки (они отображаются серым цветом).
- **Прозрачное окно** – наложение поверх любых приложений, не мешает работе.
- **Управление паузой** – кнопка в окне (или клавиша `X`) приостанавливает/возобновляет обработку жестов.
- **Отображение координат** – текущие координаты курсора выводятся в правой части нижней панели.
- **Полная настройка цвета** – зелёный скелет отслеживаемой руки, серый – остальные, красная точка в месте касания.

## 🎮 Использование

После запуска появится небольшое окно (400×300 пикселей) в левом нижнем углу экрана.

### Жесты
| Действие | Результат |
|----------|-----------|
| **Удерживать указательный и большой пальцы вместе** | Курсор следует за движением пальца (плавно) |
| **Быстрое касание (тап) тех же пальцев** | Левый клик мыши |
| **Смена руки** | Если поднести другую руку, она будет проигнорирована (серая). Управление останется за первой рукой. Если первая рука исчезнет, следующая появившаяся станет новой «главной». |

### Горячие клавиши
- `X` – приостановить / возобновить трекинг (эквивалент нажатия кнопки в окне)
- `Q` – закрыть приложение

### Интерфейс
- **Кнопка «РАБОТАЕТ» / «ПАУЗА»** – переключает состояние трекинга. При паузе жесты не обрабатываются, но скелеты рук продолжают рисоваться.
- **Координаты справа** – показывают текущие координаты курсора в формате `X, Y`.

## ⚙️ Настройка параметров

В коде (в классе `VideoThread`) можно изменить следующие константы:

```python
DIST_THRESHOLD = 30          # Расстояние между пальцами для активации жеста (в пикселях)
TAP_TIME = 0.3               # Максимальная длительность касания для клика (секунды)
SMOOTHING_ALPHA = 0.6        # Сглаживание движения (0..1): чем выше, тем плавнее, но больше запаздывание
```

Также можно изменить размер и положение окна в `MainWindow.__init__`:

```python
self.window_width = 400
self.window_height = 300
# Позиция:
self.setGeometry(10, screen.height() - self.window_height - 10, ...)
```

## 🛠️ Как это работает

1. **Захват видео** – OpenCV читает кадры с веб-камеры.
2. **Детекция рук** – MediaPipe возвращает координаты 21 ключевой точки на каждой руке.
3. **Выбор главной руки** – первая обнаруженная рука запоминается по метке (`Left`/`Right`), все остальные игнорируются для управления, но рисуются.
4. **Обработка жестов**:
   - Вычисляется расстояние между кончиками большого и указательного пальцев.
   - Если расстояние меньше порога – начинается отсчёт времени.
   - Если удержание длится дольше `TAP_TIME` – курсор перемещается согласно позиции указательного пальца.
   - Если касание было короче `TAP_TIME` – выполняется клик.
5. **Сглаживание** – применяется экспоненциальное скользящее среднее для плавного движения курсора.
6. **Отображение** – скелет руки рисуется на прозрачном холсте, который через PyQt5 выводится в окно без рамок.
7. **Пауза** – при нажатии кнопки или `X` прекращается обработка жестов, но отрисовка продолжается.

Проект тестировался на:
- Windows 11, Python 3.11
- Встроенная веб-камера 720p
